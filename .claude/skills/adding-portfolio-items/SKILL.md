# Adding Portfolio Items (codiva-site)

## Overview

The codiva.cl portfolio is a bilingual (es/en) list of projects rendered from data, not
hardcoded markup. Adding a project = describing the live site, capturing a screenshot, and
adding one object to the `projects.items` array in **both** locale files. The screenshot is a
deterministic step handled by a script; the copy is written by hand (and translated).

Run all commands from the **project root** (`codiva-site/`).

## Where things live

| File | Role |
|---|---|
| `src/i18n/es.js` → `projects.items` | **Spanish** content (edit) |
| `src/i18n/en.js` → `projects.items` | **English** content (keep in sync, same order) |
| `src/components/Portfolio.astro` | Renders the list — rarely needs changes |
| `src/styles/global.css` → `/* --- PORTFOLIO --- */` | Styling |
| `public/portfolio/*.webp` | Screenshots (~1200px wide) |
| `.claude/skills/adding-portfolio-items/scripts/capture.sh` | Screenshot → webp helper |

The section renders at `id="projects"`; the nav has a `#projects` link in `nav.links`.

## Steps to add a project

1. **Describe the site.** Visit the URL (or WebFetch it) and note: what it is, its purpose,
   main features, target audience. Write a 1–3 sentence `desc` from this.
2. **Capture the screenshot** with the helper (slug = kebab-case project name):
   ```bash
   .claude/skills/adding-portfolio-items/scripts/capture.sh https://example.com/ example-slug
   ```
   This writes `public/portfolio/example-slug.webp`. Aim for < ~120 KB (the script targets
   webp q82 @ 1200px). If the site is login-walled, the landing/hero shot is fine.
3. **Add the item to `src/i18n/es.js`** at the **top** of `projects.items` (newest first):
   ```js
   {
     name: 'Project Name',         // identical across locales
     year: '2025',                 // string
     type: 'Plataforma web',       // type OR client name — translate per locale
     tags: ['Industry', 'Capability'], // 2–3 chips, two-axis (see below) — translate
     image: '/portfolio/example-slug.webp', // same path both locales; omit for text-only
     desc: 'Una a tres frases: el problema y lo que construimos.', // translate
     url: 'https://example.com/',  // optional; omit/'' to hide the link
     linkLabel: 'Visitar sitio',   // only if url set — 'Visit site' in en
   },
   ```
4. **Add the same object to `src/i18n/en.js`** in the **same position**, with `type`, `tags`,
   `desc`, and `linkLabel` translated. `name`, `year`, `image`, and `url` stay identical.
5. **Build and verify** the locales stayed in sync:
   ```bash
   npm run build
   grep -oc 'portfolio__item"' dist/index.html dist/en/index.html   # counts MUST match
   ```
   If the two counts differ, an entry is missing or malformed in one locale.

## Field reference

| Field | Required | Notes |
|---|---|---|
| `name` | yes | Identical across locales. |
| `year` | yes | String, e.g. `'2025'`. |
| `type` | yes | Project type (`Plataforma web` / `Web platform`) or client name. Translate. |
| `tags` | yes | 2–3 strings, two-axis (see below). Translate. |
| `image` | no | Path under `public/` (e.g. `'/portfolio/slug.webp'`). Renders a screenshot above the text, clickable when `url` is set. Omit → text-only. Same path both locales. |
| `desc` | yes | 1–3 sentences: the problem + what we built. Translate. |
| `url` | no | Live/case-study link. If omitted/empty, the "Visit" link is not rendered. |
| `linkLabel` | only if `url` set | `Visitar sitio` (es) / `Visit site` (en). |

## Tag convention (two-axis)

Pair an **industry/domain** tag with a **capability** tag (optionally a third for model/scale):

- Industry: `Inteligencia de Medios`, `PetTech`, `FinTech`, `HealthTech`, `E-commerce`, `EdTech`, `Logistics`, …
- Capability: `Dashboard`, `App Móvil`/`Mobile App`, `API`, `E-commerce`, `Computer Vision`, …
- Model/scale (optional): `SaaS`, `MVP`, `Landing`, …

Examples in use: `['PetTech', 'App Móvil', 'SaaS']`, `['Inteligencia de Medios', 'Dashboard', 'SaaS']`.

## Gotchas

- **Order = display order.** New items go at the **top** of the array. The auto-numbered index
  (`01`, `02`, …) is generated by `Portfolio.astro` — never number items manually.
- **Both locales, same order.** A mismatch in count or order breaks the es/en parity.
- **No detail pages.** Each project is a single row (screenshot · name · year · type · tags ·
  description · link). Keep `desc` self-contained; the only deep link is `url`.
- **NDA / confidential clients:** use a generic `type` (e.g. `Plataforma web`) or `'Confidencial'`
  instead of a client name, and skip `url`.
- **Accents:** write real UTF-8 accents (`á`, `ñ`) directly in strings.
- Only edit the i18n files to add content — touch `Portfolio.astro`/CSS only to change layout.

## Regenerating the social share image

Unrelated to portfolio items but in the same asset pipeline: `public/og-image.png` (1200×630,
referenced by `src/layouts/Layout.astro`) is a branded card rendered via headless Chrome + sharp.
If the brand/tagline changes, re-render it (HTML card → Chrome screenshot at 2× →
`sharp.resize(1200,630)` → `public/og-image.png`). A missing file = broken link previews.
